# 1.1.0

## New features

- Added `auto_release` option to both `pypackage` and `pyservice` templates (default: `yes` for GitHub hosting). When enabled, two workflows are generated:
  - `pr-checks.yml` - runs on PRs to the release branch and blocks merges where a version bump is present but the changelog is missing or unpopulated
  - `release.yml` - runs on merge to the release branch and automatically tags the commit and creates a GitHub Release using the changelog as release notes
- The check logic lives in a shared composite action (`.github/actions/check-release-ready`) symlinked from `_shared` into both templates
- Added `scripts/release.py` and `make release` target to `pyservice`, which previously had neither
- This repo's own release process is now automated using the same workflows

## Documentation

- Rewrote `docs/pypackage/releasing.md` and `docs/pyservice/releasing.md` to share a consistent structure and cover both auto and manual release flows
- Updated `docs/pypackage/usage-github.md` and `docs/pyservice/usage-github.md` to mention the `release-ready` branch protection check
- Updated `README.md` release instructions to reflect the automated flow
