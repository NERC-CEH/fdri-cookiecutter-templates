# 1.2.0

## New features

- Added a `pre-push` git hook to the `pyservice` template (`.githooks/pre-push`). On every
push it checks whether `pyproject.toml`'s version changed; if not, it auto-bumps the patch,
regenerates `uv.lock`, commits, and re-pushes. This ensures every staging PR carries a unique
image tag and never collides with ECR's immutable tag policy. Install once with
`make install-hooks`.
- Added a `verify-version-bump` CI job to `pyservice/pipeline.yml`. Runs on PRs targeting
`staging` and fails if the PR head does not include a version bump, catching pushes made
without the hook installed and bot-created PRs (e.g. dependabot).
- Added `scripts/bump.py --no-changelog` flag (patch-only). Used internally by the pre-push
hook to create a bare bump commit without a changelog stub.
- `scripts/bump.py` now runs `uv lock` after every bump and includes `uv.lock` in the bump
commit, keeping the lockfile in sync with the version.
- Added `docs/versioning_and_releases.md` to the `pyservice` template explaining the full
patch-bump flow, manual minor/major bumps, and production release process.

## Bug fixes

- `release-ready` job in all three `pipeline.yml` files (`pyservice` template, `pypackage`
template, and this repo's own CI) previously exited 0 when no version bump was present,
silently allowing merges to the release branch without a bump. All now exit 1 with an
error message.

## Documentation

- Added `make install-hooks` step to the "Get Started" section of `CONTRIBUTING.md` (shared
template, applies to both `pyservice` and `pypackage`).
- Added a "Versioning and Releases" section to `pyservice/README.md` linking to the new docs
page, and added `make install-hooks` to the developer setup steps.
- Added `pyservice/docs/index.md` with a Technical Documentation section linking to
`versioning_and_releases.md`.
